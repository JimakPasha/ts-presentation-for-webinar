UNION TYPE

Тип объединение. Это мощный механизм позволяющий создавать из множества существующих типов данных логическое условие, по которому данные могут принадлежать к одному из указанных типов. Объединение указывается с помощью оператора прямой черты |, по обе стороны которой, располагаются типы данных.

VOID

Это значение, возвращаемое функцией, которая ничего не возвращает. Если в теле функции отсутствует оператор return или после return не указано возвращаемого значения, предполагаемым типом возвращаемого значения будет void.

В JS функция, которая ничего не возвращает, "неявно" возвращает undefined. Однако, в TS void и undefined - это разные вещи.

(кодим)

SlIDE--------------------------------------------------------------------------------------------------------

NEVER

Некоторые функции никогда не возвращают значений.
Тип never представляет значение, которого не существует. Чаще всего, это означает, что функция выбрасывает исключение или останавливает выполнение программы.

(кодим)

SlIDE--------------------------------------------------------------------------------------------------------

UNKNOWN & ANY

Представляет любое значение. Он похож на тип any, но является более безопасным, поскольку не позволяет ничего делать с неизвестным значением.
Это бывает полезным для описания типа функции, поскольку таким способом мы можем описать функцию, принимающую любое значение без использования типа any в теле функции. Другими словами, мы можем описать функцию, возвращающую значение неизвестного типа.

TYPEDEF ФАЙЛЫ

Цель файла определения типов помочь ts понять как работают сторонние js библиотеки.
Можем использовать их для npm модулей, а также для тегов скрипт.
В конце концов, он в основном используется для определения типов.

SlIDE--------------------------------------------------------------------------------------------------------

Когда мы пишем TS код, мы можем использовать библиотеки JS и можем использовать любой js код.
Но есть один момент, ts хочет проверять наш код и знать типы всего, что находится в приложении и поэтому мы должны использовать ФАЙЛЫ ОПРЕДЕЛЕНИЯ ТИПОВ. Это некая прослойка. Эти файлы говорят ts компилятору типы параметров и возвращаемые значения всех функций которые есть в библиотеке.
Иногда файлы определения типов устанавливаются автоматически, а иногда их надо специально доустановить и зачастую в популярных библиотеках они уже есть и устанавлваютя @types/[имя библиотеки]

SlIDE--------------------------------------------------------------------------------------------------------

И когда мы установили наши types. Мы можем зайти в этот файл. Название у него d.ts.
Иногда очень удобно в библиотеке использовать эти файлы, как документацию, мы можем видеть там все функции, что они принимают и что возвращают. Там нет никакой имплементации, только описание типов.
